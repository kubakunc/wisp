# Wisp audio files

The catalogue lives in `src/lib/sounds/sounds.json` — the `file` field of each
entry is the filename expected here (or under `cdn-sounds/` for remote sounds),
and `premium` controls Pro vs free.

After adding/replacing bundled files: `npm run build && npx cap sync android`.

## Bundled sounds (shipped in the APK)

Only **three** WAV files live here and are committed to the repo:

- `white-noise.wav`
- `pink-noise.wav`
- `brown-noise.wav`

These are regenerated by `node scripts/generate-sounds.mjs` (bundled → here,
rest → `cdn-sounds/`).

## Remote sounds (downloaded on first use)

The other 29 sounds are written by the generator to `cdn-sounds/` (git-ignored).
Upload `cdn-sounds/*` to the host behind `VITE_SOUND_CDN`, then set
`VITE_SOUND_CDN` in your `.env` file (e.g. `https://cdn.example.com/sounds`) to
the bucket base URL. In dev, if `VITE_SOUND_CDN` is unset the app falls back to
the local `static/` copy so development still works.

## Generated (do not hand-edit — run `node scripts/generate-sounds.mjs`)

Every catalogue sound is procedurally synthesized into a seamless-looping,
44.1 kHz / 16-bit mono **WAV** (~10 s) by `scripts/generate-sounds.mjs`. The
generator reads `sounds.json` and (re)writes one `.wav` per entry, routing
bundled sounds to `static/sounds/` and the rest to `cdn-sounds/`. These are
royalty-free, deterministic, offline-generated ambiences — not field recordings.

## Swapping in a real recording

To replace a synthesized sound with a real loop, drop the file in `static/sounds/`
(for bundled) or `cdn-sounds/` (for remote) and point that catalogue entry's
`file` at it. The generator only (re)writes entries whose `file` ends in `.wav`,
so an entry pointing at e.g. `rain.mp3` is left untouched. Real recordings
should be CC0/royalty-free, loudness-normalized (~−14 LUFS), and loop-verified
(seamless tail→head crossfade).
